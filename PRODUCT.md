# Product

## Register

product

> 注:底子是 product(日常自用的资讯工具,效率与可读性优先),但作为简历/作品集项目,
> 视觉完成度要求高于普通内部工具——截图要能独立成立。单独页面如开源 README 配套
> 页可按 brand 处理,逐任务覆盖。

## Users

- 自部署的个人开发者 / 信息工作者(首要用户 = 维护者本人)。
- 关心 AI 行业前沿、被信息过载困扰;中英双语用户均覆盖。
- 使用场景:每日早晚浏览日报与 feed、随手粘贴 URL 入库、用 Q&A Agent 检索回顾。
  长会话阅读(文章正文、深度摘要、视频转写)是核心任务,不是附属。
- 次级受众:GitHub 上看到截图、决定是否自部署的潜在用户,以及看简历的招聘方。

## Product Purpose

降低高质量信息的获取成本:聚合多平台资讯源,AI 打分过滤低质内容,生成每日精选日报,
支持粘贴 URL 入库与语义检索 Q&A。成功 = 维护者每天真实使用它替代刷信息流,
且界面截图足以作为作品集亮点。

## Brand Personality

**温和 · 学习感 · 个人化。**

像一本认真维护的手帐/文库本,而不是一台冰冷的仪表盘。「勉強(Benkyou)」的日语语源
是品牌锚点:克制的装饰、纸面的安定感、对排版的敬意。温和不等于软弱——层次清晰、
反馈明确,只是不喧哗。

正向参考(取其具体特质,不是整体模仿):

- **Notion**:中性但不冰冷的工具感,柔和的层次与交互反馈,「自己的空间」的归属感。
- **Arc / Linear**:动效、空状态、微交互的细节打磨——手艺感是作品集的核心证据。
- **日系纸质/文具**:克制的装饰、纸面质感、手帐/文库本的意象(由排版与质感承载,
  不是默认米色背景)。
- **Readwise Reader 类阅读态**(由长阅读场景推得):正文排版为王,阅读时界面退到背景。

## Anti-references

- **SaaS 营销风**:渐变大字、玻璃拟态、英雄区指标卡——典型 AI 生成 landing page 味。
- **陈旧后台管理风**:Ant Design 默认蓝、表格堆满、毫无排版层次的 admin 模板感。
- **过度极简留白风**:为了好看牺牲信息密度,一屏只显示三条内容。
- **信息流社交风**(由产品目的推得):本产品存在的意义就是对抗注意力劫持,
  界面不得复制无限刺激的模式。

## Design Principles

1. **阅读优先,界面退后**:feed 与详情页的一切装饰让位于正文可读性;
   阅读中浏览器外壳之外的 UI 应当几乎消失。
2. **密度即尊重**:用户来处理信息,不是来欣赏留白。一屏内呈现足量内容,
   靠排版层次(而非空白)区分主次。
3. **双语是一等公民**:中英混排(字体栈、行高、标点、字重)按 AA 之上的标准对待;
   任何排版决策都要在 zh 和 en 两种 locale 下验证。
4. **手艺感藏在细节里**:空状态、加载骨架、错误提示、微动效——这些"没人逼你做好的
   地方"做到位,是作品集叙事的核心。
5. **状态透明,语气平静**:pipeline 失败、转写进行中、源断流等系统状态要主动可见,
   但用平静的语气呈现(横幅、徽标),不用红色警报轰炸。

## Accessibility & Inclusion

- **WCAG 2.1 AA**:正文对比度 ≥4.5:1,大字 ≥3:1;全键盘可达(Cmd+K 已是 spec 承诺);
  `prefers-reduced-motion` 下所有动效有降级方案。
- **双语排版作为无障碍要求对待**:中英混排的字体回退、行高、断行不是美化项,
  是可读性底线。
- 深色模式:`prefers-color-scheme` 自动 + 设置页手动覆盖(spec §9.3),两套主题
  均须独立通过对比度检查。
